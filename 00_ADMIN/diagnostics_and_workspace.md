## 5. Правила названия файлов

1. Использовать понятные названия.
2. Не использовать названия вроде `новый файл`, `финал2` или `последний финал`.
3. Использовать английские буквы.
4. Использовать нижнее подчёркивание вместо пробела.
5. Добавлять номер версии только при необходимости.

Хорошие примеры:

`diagnostics_and_workspace.md`

`day_01_folder_structure.png`

`mobile_app_wireframe_v01.png`

Плохие примеры:

`новый документ.txt`

`финал финал 2.png`

`Безымянный-1.fig`

## 6. Правила evidence

Evidence — это проверяемое доказательство работы.

Evidence может быть:

- ссылка на Figma;
- ссылка на GitHub;
- commit;
- скриншот;
- файл;
- результат теста;
- наблюдение;
- измерение;
- цитата реального участника исследования.

Evidence не является:

- моё мнение без доказательств;
- ответ AI;
- выдуманное интервью;
- выдуманный результат теста;
- предполагаемый рост продаж;
- предполагаемый business impact.
